# Archivo del Repositorio

Esta carpeta contiene material preservado que ya no forma parte del camino activo para avanzar el `app`.

## Criterio de archivo

Se archiva aquí lo que cumple una o más de estas condiciones:

- documentación táctica o puntual de infraestructura,
- fixes históricos,
- proyectos específicos no canónicos,
- scripts de operación legacy fuera del roadmap actual,
- prototipos o presentaciones que no pertenecen al producto activo.

## Estructura

- `root-guides/`: guías sueltas de setup, testing, Docker y fixes.
- `projects/`: documentos de proyectos puntuales y propuestas no canónicas.
- `scripts/docker-legacy/`: scripts root-level de operación Docker y testing ad hoc.
- `public-legacy/`: HTMLs o piezas visuales fuera de la experiencia activa del `app`.

## Activo vs archivado

La ruta activa del repo para producto y fases del `app` ahora es:

- `base/`
- `app/`
- `docs/`
- `archive/`

`archive/` no debe usarse como fuente primaria de implementación, salvo consulta histórica.
