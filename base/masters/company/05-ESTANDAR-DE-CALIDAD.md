# Estándar de Calidad Profesional: Manual de Excelencia Live Developer

Este documento define la barra de calidad que todo entregable (generado por humanos o IA) debe superar antes de llegar al cliente. No buscamos "cumplir", buscamos **"WOW"**.

---

## 1. El ADN de la Marca: "Razón + Emoción"
Todo contenido debe equilibrar las dos dimensiones de Live Developer:
*   **La Emoción:** Narrativa, estética cinematográfica, psicología del consumidor, ganchos emocionales.
*   **La Razón:** Factibilidad técnica, métricas (ROAS, CTR), infraestructura escalable, orden y lógica.

> **Regla de Oro:** "Si es hermoso pero no convierte, es arte, no es Live Developer. Si convierte pero es feo, es spam, no es Live Developer."

---

## 2. Checklists de Calidad por Área

### A. Estrategia y Funnels
- [ ] **Nivel de Consciencia:** ¿El copy está adaptado a si el usuario conoce el problema o ya busca la solución?
- [ ] **Factibilidad:** ¿La estructura sugerida puede ser construida por nuestro equipo de desarrollo hoy?
- [ ] **Fricción:** ¿Hemos eliminado todos los pasos innecesarios en el camino del usuario?

### B. Producción Audiovisual (Cine/Foto/Video)
- [ ] **Óptica Real:** ¿El prompt evita el 'look AI' (oversaturated/plastic) y usa parámetros de lentes reales (35mm, 85mm)?
- [ ] **Iluminación:** ¿Se especifica una intención de luz (Rembrandt, Volumétrica, Golden Hour)?
- [ ] **Raccord (Continuidad):** En secuencias, ¿coinciden la ropa, el clima y la iluminación?

### C. Guiones y Copywriting
- [ ] **Regla de los 3 Segundos:** ¿El hook es tan potente que detiene el scroll?
- [ ] **Ritmo:** ¿Hay variedad en la longitud de las frases? (Frases cortas para impacto, largas para fluidez).
- [ ] **Subtexto:** ¿El diálogo suena natural o parece un folleto de ventas?

---

## 3. Protocolo de Entrega (Formato Estándar)
Para que el equipo de 8 personas trabaje al unísono, todo documento de cara al cliente debe:
1.  **Encabezado:** Logo de Live Developer + Fecha + Nombre del Proyecto/Cliente.
2.  **Objetivo:** Una sola frase sobre qué espera lograr este documento.
3.  **Cuerpo:** Organizado en Markdown limpio con headers claros.
4.  **Próximos Pasos:** Siempre terminar con "Acción Requerida" para el cliente o el equipo técnico.

---

## 4. El Filtro Final: "El Agente Auditor"
Nadie envía nada sin pasarlo antes por el **Agente Auditor de Calidad** (ver `/Agentes/AuditorCalidad.md`). Su función es criticar y destruir las debilidades del entregable para fortalecerlo.

---

> [!IMPORTANT]
> La calidad no es un acto, es un hábito. Si un agente IA genera algo mediocre, tu deber como humano es iterar el prompt hasta llegar a la excelencia. No aceptamos el primer resultado por defecto.
