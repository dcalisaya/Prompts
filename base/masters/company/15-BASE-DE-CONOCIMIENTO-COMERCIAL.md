# Base de Conocimiento Comercial y Operativa

Este documento sirve para preguntas internas frecuentes, criterios de venta y consistencia de respuesta.

## FAQ interna

### Cuando usar `COM-001`
Cuando una persona necesita responder que servicio aplica o como orientar a un cliente.

### Cuando usar `COM-002`
Cuando ya existe una necesidad concreta y hay que estructurar una proforma o alcance preliminar.

### Cuando no deberia salir una cotizacion final
- cuando el brief esta incompleto,
- cuando el servicio no existe en la matriz,
- cuando el pricing sigue pendiente,
- cuando operacion no valido factibilidad.

### Como se vende una solucion compleja
Separando los componentes por servicio y explicando el orden de ejecucion.

### Como manejar algo que el catalogo no cubre
Se marca como requerimiento especial y se escala a revision interna.

## Casos tipicos

### Caso 1: cliente pide video por dos dias en Quito
Salida esperada:
- `AV-005` Video Corporativo
- `AV-001` Grabacion en Quito
- posibles complementos: `AV-004`, `AV-007`, `AV-008`, `AV-011`

### Caso 2: cliente pide pauta Spotify para multiples episodios
Salida esperada:
- `MK-004` Pauta Digital
- posible complemento creativo con `ADS-001`
- si faltan piezas, revisar `AV-010`, `AV-011` o `AV-013`

### Caso 3: cliente quiere una landing para captacion
Salida esperada:
- `DEV-001` Landing Page
- posible complemento `MK-002` Estrategia Digital
- posible complemento `MK-004` Pauta Digital

## Objeciones frecuentes

### "Solo quiero un precio rapido"
Responder con estructura preliminar y aclarar que el valor final depende del alcance.

### "Quiero todo incluido"
Desglosar servicios y explicar que el paquete debe definirse por componentes.

### "Necesito para ya"
Aplicar validacion de urgencia y recargo si corresponde.

## Regla de madurez operativa

Si la misma pregunta aparece varias veces, debe convertirse en:

- una plantilla,
- una regla comercial,
- o un nuevo prompt operativo.
