# Estándar de Comunicados de Prensa
## Formato, Estructura y Protocolo de Distribución

Este estándar establece el formato periodístico profesional para la redacción y distribución de comunicados de prensa. Es la fuente de verdad para el agente PRSpecialist y el prompt PR-001.

---

## 1. ¿Cuándo Emitir un Comunicado de Prensa?

### 1.1 Eventos que Justifican un Comunicado

| Tipo | Ejemplo | Prioridad |
|:-----|:--------|:---------|
| **Lanzamiento** | Producto, servicio, filial, oficina nueva | Alta |
| **Alianza** | Partnership, fusión, adquisición | Alta |
| **Logro** | Premio, certificación, reconocimiento | Media |
| **Evento** | Participación en feria, summit, patrocinio | Media |
| **Posicionamiento** | Contribución a debate público | Media-Baja |
| **Crisis (en ciertos casos)** | Solo si beneficia activamente a la marca | Según caso |

### 1.2 Eventos que NO Justifican un Comunicado

- Notas internas o administrativas sin interés público
- Anuncios personales de empleados
- Promociones comerciales corrientes
- Reestructuraciones internas no estratégicas
- Actividades rutinarias sin relevancia periodística

## 2. Estructura del Comunicado (Pirámide Invertida)

```
FORMA CANÓNICA DE UN COMUNICADO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

                    ┌─────────────────────────────┐
                    │  [LOGO DE LA EMPRESA]       │
                    │  COMUNICADO DE PRENSA       │
                    │  Para publicación inmediata  │
                    └─────────────────────────────┘

ENCABEZADO (City, Date) — La ciudad y la fecha de emisión.
El primer párrafo responde: ¿Qué? ¿Quién? ¿Cuándo? ¿Dónde? ¿Por qué?

CUERPO PRINCIPAL (Lead) — Primero lo más importante.
- Párrafo 1: Las 5W en 3-4 líneas (qué, quién, cuándo, dónde, por qué).
- Párrafo 2-3: Contexto, datos de soporte, cita(s) de spokesperson(s).

CUERPO SECUNDARIO — Detalles adicionales.
- Datos específicos, cifras, antecedentes de la empresa.
- Más citas si hay múltiples voceros relevantes.

CUERPO DE CONTEXTO — Solo si es necesario.
- Información de fondo que contextualiza pero no es crítica.

BOILERPLATE — La empresa en un párrafo.
- Quién es, qué hace, desde cuándo, en qué mercados.

INFORMACIÓN DE CONTACTO — Datos del periodista de contacto.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 3. Reglas de Escritura Profesional

### 3.1 El Lead: Las 5W en 3-4 Líneas

El primer párrafo es lo único que muchos periodistas leerán. Debe contener:

| Elemento | Pregunta | Ejemplo |
|:---------|:--------|:--------|
| **Qué** | ¿Qué ocurrió? | "Live Developer lanza su servicio de producción audiovisual en Ecuador" |
| **Quién** | ¿Qué organización? | "Live Developer, agencia de marketing digital con sede en Quito" |
| **Cuándo** | ¿Cuándo? | "a partir del próximo mes de marzo" |
| **Dónde** | ¿Dónde? | "en todo el territorio ecuatoriano" |
| **Por qué** | ¿Para qué / por qué importa? | "para atender la creciente demanda de contenido digital de marcas en el país" |

**Formato**:
> **[ORGANIZACIÓN]**, [ciudad] — [Fecha] — [Párrafo de lead con las 5W en 3-4 líneas].

### 3.2 Vocería: Cómo Insertar Citas

Las citas deben aportar perspectiva humana que no esté en el lead.

```
FORMATO DE CITA:
" [Cita real del spokesperson, no genérica, en primera persona] ",
dijo [Nombre], [Cargo del spokesperson].

REGLAS:
• La cita SIEMPRE es del spokesperson, no del redactor del comunicado.
• La cita debe aportar información o perspectiva nueva, no repetir el lead.
• Máximo 2-3 citas por comunicado.
• Incluir al menos una cita de un cargo executive (CEO, Director, Gerente).
• El cargo debe decir qué representa, no solo quién es.
```

**Ejemplo de cita buena:**
> "Ecuador merece contenido de clase mundial. Esta alianza nos permite ofrecer a las marcas ecuatorianas una producción audiovisual al nivel de cualquier agencia regional", dijo **[Nombre]**, **[Cargo]** de **[Empresa]**.

**Ejemplo de cita mala (prohibida):**
> "Estamos muy emocionados de anunciar esto", dijo el gerente.

### 3.3 Lenguaje Prohibido en Comunicados

| Prohibido | Razón | Alternativa |
|:----------|:------|:------------|
| "Somos líderes en..." | Autoelogio sin respaldo | "Somos la agencia con [dato verificable]" |
| "Lo mejor del mercado" | Subjetivo | "Ofrecemos [característica específica]" |
| "Sin precedentes" | Exageración | "Por primera vez en Ecuador..." |
| "Solución revolucionaria" | Cliché | "Una nueva herramienta para..." |
| Ataques a competidores | Peligroso legalmente | Centrarse en propuesta propia |
| Datos no verificados | Riesgo legal | Solo cifras auditadas |

### 3.4 Datos y Cifras

- Toda cifra debe tener fuente o contexto. Si se dice "líder", demostrarlo.
- Usar números concretos: "20%" no "veinte por ciento" en el lead.
- Solo incluir años en contexto: "desde 2018" no "fundada hace 7 años".

## 4. Tipos de Comunicado según Evento

### 4.1 Lanzamiento de Producto/Servicio

```
ESTRUCTURA:
1. Lead: Qué se lanza, para quién, cuándo
2. Contexto: Por qué existe la necesidad
3. Características: 3-4 features clave (en bullets o párrafo)
4. Precio/disponibilidad: Si aplica
5. Cita del executive: Visión estratégica
6. Cita del cliente early adopter: Si existe
7. Boilerplate
```

### 4.2 Alianza / Partnership

```
ESTRUCTURA:
1. Lead: Quién se une con quién, para hacer qué
2. Contexto: Por qué ambas partes se benefician
3. Qué aporta cada parte: Claro, no ambiguo
4. Timeline: Cuándo entra en vigencia
5. Cita de cada organization (máximo 2-3)
6. Boilerplate de ambas organizaciones
```

### 4.3 Logro / Premio

```
ESTRUCTURA:
1. Lead: Quién ganó qué, de quién
2. Qué es el premio/reconocimiento: Credenciales del otorgante
3. Por qué se ganó: Criterio evaluado
4. Impacto: Qué significa para la organización
5. Cita del líder
6. Boilerplate
```

## 5. Boilerplate Estandarizado

El boilerplate es el único párrafo que se repite en todos los comunicados. Debe estar siempre actualizado.

```
FORMATO BOILERPLATE:
[Nombre de Empresa] es [descripción breve: qué hace, para quién, desde cuándo].
Con [dato de tamaño/scope: número de clientes, presencia, equipo],
[empresa] se ha posicionado como [diferenciador breve]. Con sede en [ciudad],
opera en [mercados] y se especializa en [áreas clave]. Más información en [web].

EJEMPLO:
Live Developer es una agencia de marketing digital y producción audiovisual con
sede en Quito, fundada en 2018. Con un equipo de 15 profesionales y más de 80
proyectos completados, la agencia se especializa en estrategia digital,
producción de contenido y soluciones de performance marketing para marcas en
Ecuador y Latinoamérica. Más información en www.livedeveloper.ec
```

## 6. Distribución

### 6.1 Base de Datos de Medios

La distribución requiere una base segmentada por:

```
SEGMENTACIÓN DE BASE DE DATOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ GENERALISTAS NACIONALES: medios de alcance nacional
□ ESPECIALIZADOS POR INDUSTRIA: según el tema del comunicado
□ REGIONALES/LOCALES: si el evento tiene impacto geográfico específico
□ BLOGUEROS/INFLUENCERS DE PRENSA: journalists individuales en redes
□ INTERNACIONALES: si es relevante para medios fuera de Ecuador
□ SECTORIALES: según industria del cliente
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 6.2 Protocolo de Distribución

```
ANTES DE ENVIAR:
□ Revisar ortografía y gramática (mínimo 2 ojos)
□ Verificar que todos los datos sean correctos
□ Obtener aprobación del spokesperson cited
□ Obtener aprobación de legal (si hay datos sensibles)
□ Formatear con logo, fuentes y espaciado profesionales
□ Preparar versión HTML y versión plain text
□ Adjuntar foto/activos visuales si corresponde

ENVÍO:
□ Enviar con mínimo 5 días de anticipación para eventos
□ Enviar temprano en la semana (martes-jueves, 8-10am)
□ Usar campo "Para:" visible (no BCC masivo)
□ Personalizar línea de asunto: [EMPRESA] - [TITULAR NOTICIOSO]
□ Incluir medios visuales en el cuerpo del email

SEGUIMIENTO:
□ Llamada de seguimiento 24-48h después a medios clave
□ Medir cobertura: tracking de menciones
□ Documentar resultados para caso de éxito
```

### 6.3 Línea de Asunto

| Tipo | Ejemplo |
|:-----|:--------|
| Bueno | "Live Developer y Grupo Bolívar sellan alianza para transformación digital" |
| Bueno | "Qodee Studio lanza primera plataforma de e-learning para PyMEs ecuatorianas" |
| Malo | "Comunicado de prensa de nuestra empresa" |
| Malo | "URGENTE" (solo si es genuinamente urgente) |

## 7. Errores Frecuentes

| Error | Consecuencia |
|:------|:-------------|
| Lead sin noticias | El periodista lo borra sin leer |
| Cita genérica | Descalifica la nota |
| Ataque a competidores | Riesgo legal, falta de profesionalismo |
| Falta de dato de contacto |就无法 hacer seguimiento |
| Formato de email como spam | Llega a papelera |
| Enviar solo el PDF | Debe enviarse en el cuerpo del email |
| No enviar a medios especializados | Pierde relevancia en el sector |
| Boilerplate desactualizado | Información incorrecta en medios |

---

## 8. Checklist Pre-Envío

- [ ] ¿Es esto realmente una noticia?
- [ ] ¿El lead responde las 5W?
- [ ] ¿Hay al menos una cita de un spokesperson real?
- [ ] ¿Los datos son verificables?
- [ ] ¿El boilerplate está actualizado?
- [ ] ¿Hay datos de contacto correctos?
- [ ] ¿La base de medios está segmentada correctamente?
- [ ] ¿Se obtuvo aprobación antes de enviar?
- [ ] ¿El formato es profesional (logo, fuentes, espaciado)?
- [ ] ¿Se incluyen activos visuales?
