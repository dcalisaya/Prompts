# Roadmap del App por Fases

## 1. Proposito

Este roadmap conserva solo la fase activa y la capa base de contratos del producto.

La documentacion historica de fases 1 a 5 fue retirada de `docs/` para dejar la carpeta enfocada en el estado actual y en los siguientes bloques de evolucion.

## 2. Fase 0. Base de Contratos y Producto

Objetivo:

- asegurar que el `app` tenga una base documental y estructural estable.

Incluye:

- vision del producto
- arquitectura funcional
- modulos y pantallas
- metadata normalizada
- validacion de `base/masters`
- `base/json` consistente

## 3. Fase 6. Estado Expandido y Plataforma

Objetivo:

- llevar el `app` desde un workspace operativo usable hacia una capa de plataforma con estado expandido, ownership y preparacion de integraciones.

Bloques activos:

### Bloque 6A. Modelo de estado expandido

- transiciones mas ricas
- estados derivados
- visibilidad de estado a nivel de trabajo

### Bloque 6B. Ownership y permisos basicos

- asignacion de responsables
- visibilidad de ownership
- restricciones minimas por accion

### Bloque 6C. Preparacion de integraciones futuras

- contratos para integraciones
- fronteras de plataforma
- compatibilidad con servicios externos

## 4. Nota

La eliminacion de las fases 1 a 5 de esta carpeta no cambia el estado real del producto implementado.

Solo reduce el volumen documental activo para enfocar la siguiente etapa.
