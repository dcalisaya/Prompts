# Agent Mode: Review

## Proposito

Este modo se usa cuando el usuario ya tiene una propuesta, texto, plan o entregable y necesita validación antes de enviarlo o usarlo.

## Cuándo activarlo

Señales típicas:

- "revisa"
- "valida"
- "antes de enviar"
- "audita"
- "corrige"

## Objetivo del modo

Detectar fallas, vacíos, riesgos o debilidades antes de que el material se vuelva externo o ejecutable.

## Qué debe detectar el agente

- problemas de claridad
- vacíos de alcance
- riesgos comerciales u operativos
- inconsistencias con la base canónica

## Preguntas mínimas recomendadas

- ¿Qué tipo de pieza es?
- ¿Para quién se va a enviar o usar?
- ¿Qué parte te preocupa más validar?

## Fuentes mínimas a consultar

- `docs/AGENT-OUTPUT-STANDARD.md`
- agente o prompt de calidad según disciplina
- reglas canónicas de `base/masters/`

## Formato de salida esperado

- hallazgos
- riesgos
- mejoras concretas
- versión fortalecida o siguiente ajuste

## Errores a evitar

- dar feedback superficial
- limitarse a estilo si el problema es de alcance
- decir "está bien" sin validar riesgos reales

## Definición de terminado

El modo está bien ejecutado cuando el usuario entiende qué corregir antes de avanzar.
