# Agent Learnings

## Proposito

Este documento registra aprendizajes persistentes sobre cómo operan los agentes dentro de este workspace.

No es una bitácora de conversación.

Es una memoria de patrones útiles para refinar:

- protocolo
- intake
- output
- starter prompt
- y, más adelante, la base canónica

## Regla de uso

Solo deben registrarse aprendizajes que:

- se repiten
- afectan la utilidad operativa
- justifican una regla nueva o un ajuste claro

## Formato recomendado

Cada aprendizaje debería incluir:

- fecha
- tipo de caso
- modo
- hallazgo
- riesgo detectado
- regla derivada
- impacto esperado

## 2026-03-23

### Proformas breves tienden a sobreasunción de alcance

- tipo de caso: proforma breve
- modo: `quote`
- contexto: solicitud simple para describir renovación de sitio web en WordPress

Hallazgo:

- varios agentes entienden bien el repo y el modo `quote`
- pero tienden a sobreasumir componentes técnicos no solicitados

Sobreasunciones frecuentes detectadas:

- SEO técnico
- analítica web
- seguridad
- migración completa
- hosting o infraestructura
- cantidad de páginas
- capacitación
- cronogramas cerrados

Riesgo detectado:

- la respuesta deja de ser una descripción preliminar y empieza a parecer un alcance cerrado no validado

Regla derivada:

- en solicitudes `quote` breves, el agente debe priorizar una descripción corta, preliminar y comercial
- no debe inferir componentes técnicos o entregables adicionales salvo que el usuario los mencione o los pida
- si necesita ampliar, debe hacerlo como supuesto o bloque opcional, no como alcance cerrado

Impacto esperado:

- propuestas más disciplinadas
- menos invención de alcance
- mejor utilidad para operadores comerciales
